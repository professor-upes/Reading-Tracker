Environments (Production vs Testing/Local)

Recommended workflow:
  • Production: hosted/official version (stable)
  • Testing: a separate branch or folder (staging)
  • Local: experimental sandbox on your machine
In the Product Backlog, set Environment = Testing/Local for ideas you want to test before promoting to Production.
Local-first: everything is stored in localStorage. Use Export/Import to move data between machines.
